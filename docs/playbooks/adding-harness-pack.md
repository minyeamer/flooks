# Playbook: Adding a harness pack

1. Describe the pack's domain, target users, and allowed tools.
2. Provide prompt fragments, glossary terms, and any output transformation rules.
3. Confirm that the pack does not weaken dataset or workspace authorization.
4. Add evaluation prompts that cover both valid and denied access cases.
5. Update `docs/architecture/ai-harness.md` if the pack introduces a new extension point.
